1. O que é um repositório no Git e qual a sua importância?

R: Repositório do git nada mais é do que o próprio nome diz, é um repositório
de arquivos, e versionamento, local para visualizar alterações feitas no arquivo
ao longo de suas versões, dessa maneira caso algo de errado em alguma 
atualização do projeto pode-se visualizar o arquivo antes 
das alterações feitas.

2. Qual a função do Git Bash?

R: Git Bash é um terminal para sistemas Windows de comandos em linha. Isso tudo porque
o Linux e o MAC utiliza o sistema unix, basicamente, o git bash emula o unix no terminal
para realizar a integração no Windows.

3. O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?

R: O git add informa ao Git que você deseja incluir atualizações em um arquivo
específico no próximo commit. O git add <arquivo> informa um arquivo só o git add .
adiciona todos os arquivos do diretório.

4. Por que utilizamos o ponto (.) no comando git add .?

R: Adiciona todos os arquivos presente naquele diretório

5. Explique o comando git commit e por que é importante criar uma mensagem descritiva.

R: O git commit é um comando onde o Git realiza uma atualização descritiva do projeto
é importante escrever uma boa descrição da atualização para que todos os usuarios que
visualizarem o projeto entenderem o que foi feito naquela atualização.

6. Qual a função do comando git push?

R: o comando git push executa a função de enviar o diretório que foi feito o commit
para o repositório online.

7. Qual a importância de criar commits frequentes durante o desenvolvimento de um projeto?

R: Para que fique resgistrado o que você alterou no projeto, muito importante para projetos
que envolvem varias pessoas. 

8. Explique a função das tags HTML e como elas estruturam uma página.

R: Existem n tipos de tags HTML, elas servem para informar o navegador a estrutura do site.

9. O que são atributos em uma tag HTML e qual a sua função?

R: Os atributos são responsaveis a informar ao navegador qual a importância para a 
estrutura do site, como o title, o title é um elemento muito importante e pode 
ser uma peça chave para a busca no navegador.

10. O que é o Emmet e como ele facilita o desenvolvimento de páginas HTML?

R: Emmet é um plugin (para diversos editores/IDEs) que facilita bastante a escrita 
do HTML

11. O que é o box-sizing no CSS e por que usamos ele no reset inicial?

R: Box sizing define como a largura e a altura de um elemento são calculadas,
usamos para normalizar o comportamento entre browsers, para deixar todos
iguais inicialmente e a partir dai começar as alterações.


12. Por que é importante utilizar tags semânticas no HTML?

R:As tags semânticas são importantes, pois tornam as paginas web mais
acessíveis, compreensíveis e otimizadas para mecanismos de buscas,
além de garantir acessibilidade para pessoas com deficiencia visuais
ou cognitivas.

13. O que são media queries no CSS e qual a sua função?

R: No CSS, media queries é um recurso que vai dar responsividade ao seu
site, permitindo que ele seja formatado dependendo do tipo de tela
que o usuário esta usando.

14. Qual a função do GitHub Pages?

R: GitHub Pages é uma ferramenta dentro do GitHub que permite deixar
seu projeto em um site para que outras pessoas que possuem o link
acessar seu projeto.

15. O que é o Flexbox no CSS e para que ele é utilizado?

R: Flexbox é um modelo de pagina que permite organizar os elementos
em containers de forma flexivel.

16. Qual a diferença entre justify-content e align-items em um container Flexbox?

R:
 justify-content: é uma propriedade que define o alinhamento dos elementos
no eixo principal de um elemento com propriedade flex.

 align-items: define como os elementos são alinhados no eixo transversal
de um elemento com a propriedade flex.

justify-content define o alinhamento de um item dentro de um elemento 
no eixo principal enquanto o align_items define como os elementos são alinhados.


17. Como o flex-grow funciona e qual o seu propósito no layout Flexbox?

R: Flex-grow é uma propriedade do flexbox que controla o crescimento de um item
dentro de um elemento flexivel.

18. Explique a diferença entre flex-direction: column; e flex-direction: row;.

R: (flex-direction: column;): deixa o layout da pagina em coluna com um item em baixo do outro.
   (flex-direction: row;): deixa o layout da pagina em linha com um item do lado do outro.
